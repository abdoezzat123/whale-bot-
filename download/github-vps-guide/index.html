1️⃣ تجهيز ملفات المشروع على جهازك

عندك ملفات البوت في فولدر واحد. تأكد إن فيه الملفات دي:

telegram-whale-bot/
├── bot.py                  # الكود الرئيسي
├── bsc_monitor.py          # مراقب BSC
├── whales.py               # محافظ الحيتان
├── deploy.sh               # سكريبت التحكم
├── Dockerfile              # إعدادات Docker
├── docker-compose.yml      # يشغل البوتين
├── requirements.txt        # متطلبات Python
├── .env                    # الإعدادات (سرّي!)
├── .dockerignore
└── README.md
مهم جداً: ملف .env فيه الـ Telegram Bot Token و Chat ID - ده سرّي! لازم نضيفه لـ .gitignore علشان ما يترفعش على GitHub.
1 إنشاء ملف .gitignore

افتح Terminal/CMD في فولدر المشروع واكتب:

# إنشاء ملف .gitignore
cat > .gitignore << 'EOF'
.env
*.db
data/
__pycache__/
*.pyc
venv/
.venv/
.vscode/
.idea/
.DS_Store
*.log
EOF

أو أنشئه يدوياً بأي محرر نصوص وحط فيه المحتوى فوق.

2️⃣ إنشاء حساب GitHub (لو معندكش)

لو عندك حساب GitHub، اتخطى للقسم 3. لو لأ:

1 افتح GitHub وسجل

ادخل على: https://github.com/signup

  • الإيميل
  • باسورد قوي
  • username (هتستخدمه في الـ repo URL)

هيوصلك كود تحقق على الإيميل - أكد الحساب.

3️⃣ إنشاء repository جديد

1 افتح GitHub ودوس "New"

بعد ما تسجل دخول:

  • دوس على + فوق على اليمين
  • اختر "New repository"
صفحة إنشاء repository جديد على GitHub
2 املأ بيانات الـ repo
  • Repository name: whale-bot
  • Description: Solana + BSC whale tracker bot (اختياري)
  • Visibility: اختار Private (مهم! علشان الكود سرّي)
  • دوس "Create repository"
اختار Private مش Public! الكود فيه الـ bot token والـ chat ID لو اترفع خطأ.

4️⃣ توليد Personal Access Token (PAT)

علشان ترفع الكود من Terminal/CMD، GitHub محتاج token بدل الباسورد.

1 افتح Settings
  • دوس على صورتك فوق على اليمين
  • اختر Settings
  • انزل تحت واختر Developer settings (آخر خيار)
  • اختر Personal access tokens → Tokens (classic)
  • دوس "Generate new token (classic)"
صفحة توليد Personal Access Token
2 اضبط الـ token
  • Note: whale-bot-deploy
  • Expiration: 90 days (أو No expiration)
  • Scopes: علم على ✓ repo (كل الـ sub-options)
  • دوس "Generate token" في آخر الصفحة
انسخ الـ token فوراً! مش هتقدر تشوفه تاني بعد كده. الـ token شكله زي: ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

5️⃣ رفع الملفات على GitHub

عندك طريقتين: Git CLI (للمتقدمين) أو الرفع من المتصفح (للمبتدئين).

🅰️ الطريقة الأولى: الرفع من المتصفح (الأسهل)

1 افتح الـ repo اللي عملته

هتلاقي زرار "uploading an existing file" - دوس عليه.

صفحة رفع الملفات على GitHub
2 اسحب كل الملفات

اسحب كل الملفات من فولدر المشروع (ما عدا .env و data/) للمنطقة المنقطة.

أو دوس "choose your files" واختارها.

بعد ما ترفع كل الملفات، دوس "Commit changes" تحت.

مترفعش ملف .env! ده فيه الـ Bot Token و Chat ID. هنظبطه على الـ VPS مباشرة.

🅱️ الطريقة الثانية: Git CLI (الأسرع والأنظف)

1 تثبيت Git (لو مش متثبت)

Windows: نزله من https://git-scm.com

Mac: brew install git

Linux: sudo apt install git

2 إعداد Git مرة واحدة 
git config --global user.name "اسمك"
git config --global user.email "إيميلك@example.com"
3 رفع المشروع

افتح Terminal/CMD في فولدر المشروع واكتب:

# 1. تهيئة git
git init

# 2. إضافة كل الملفات (ما عدا اللي في .gitignore)
git add .

# 3. أول commit
git commit -m "Initial commit - whale bot"

# 4. تحديد الـ branch
git branch -M main

# 5. ربط بالـ repo (غير USERNAME باسمك)
git remote add origin https://github.com/USERNAME/whale-bot.git

# 6. رفع الكود
git push -u origin main

أول مرة هيطلب منك username و password. الـ username هو GitHub username، والـ password هو الـ Token اللي عملته في القسم 4.

لو ظهرلك خطأ Authentication failed: استخدم الـ Token مش الباسورد في خانة الـ password.

6️⃣ الاتصال بالـ VPS بـ SSH

دلوقتي البوت على GitHub. لازم نروح للـ VPS وننزله هناك.

1 افتح Terminal

Mac/Linux: افتح Terminal

Windows 10+: افتح PowerShell أو CMD

Windows قديم: استخدم PuTTY

الاتصال بـ SSH على Terminal
2 اتصل بالـ VPS

AWS EC2 (Ubuntu):

ssh -i ~/Downloads/your-key.pem ubuntu@YOUR_PUBLIC_IP

AWS EC2 (Amazon Linux):

ssh -i ~/Downloads/your-key.pem ec2-user@YOUR_PUBLIC_IP

Oracle Cloud:

ssh -i ~/Downloads/ssh-key-xxxx.key ubuntu@YOUR_PUBLIC_IP

أول مرة هيطلب منك تأكيد - اكتب yes واضغط Enter.

الـ prompt هيغير لـ ubuntu@ip-xxx-xx-xx-xx:~$ - ده معناه إنك اتصلت بنجاح.

7️⃣ تنزيل المشروع على الـ VPS (git clone)

1 ثبت Git و Docker 
# تحديث النظام
sudo apt update && sudo apt upgrade -y

# تثبيت Git
sudo apt install -y git

# تثبيت Docker
curl -fsSL https://get.docker.com | sudo sh

# إضافة المستخدم لمجموعة docker (علشان ما نستخدمش sudo كل مرة)
sudo usermod -aG docker $USER

# تفعيل التغيير
newgrp docker
2 تنزيل المشروع من GitHub

لو الـ repo Private (موصى به):

# تنزيل المشروع (ستبدل USERNAME و TOKEN)
git clone https://USERNAME:TOKEN@github.com/USERNAME/whale-bot.git

# مثال:
git clone https://ayaodo:ghp_xxxxx@github.com/ayaodo/whale-bot.git

# الدخول للفولدر
cd whale-bot

لو الـ repo Public:

git clone https://github.com/USERNAME/whale-bot.git
cd whale-bot
تنزيل المشروع على الـ VPS
طريقة تانية للـ Private repo بدون token في URL: 
git clone https://github.com/USERNAME/whale-bot.git
# هيطلب username = GitHub username
# password = الـ Token

9️⃣ تشغيل البوت بـ Docker

1 تشغيل البوت 
# تأكد إنك في فولدر المشروع
cd ~/whale-bot

# تشغيل البوت (solana + BSC) في الـ background
docker compose up -d --build
تشغيل البوت بـ Docker

أول مرة هتاخد شوية (2-5 دقايق) علشان يعمل build للـ image. بعدها هيبقى سريع.

2 شوف الـ logs للتأكد 
# شوف الـ logs live
docker compose logs -f

# أو logs لخدمة معينة
docker compose logs -f whale-bot-solana
docker compose logs -f whale-bot-bsc

لازم تلاقي رسالة زي:

whale-bot-solana  | 🚀 البوت اشتغل!
whale-bot-solana  | 🐋 محافظ متابعة: 10
whale-bot-solana  | 💰 حد الشراء: $500

whale-bot-bsc     | 🚀 مراقب BSC اشتغل!
whale-bot-bsc     | 🐋 محافظ BSC متابعة: 3

وكمان هتلاقي رسالتين على تيليجرام بتأكد إن البوتين اشتغلوا.

3 الخروج من الـ VPS

اضغط Ctrl+C للخروج من الـ logs (البوت يفضل شغال في الـ background).

اكتب exit للخروج من الـ VPS.

البوت هيفضل شغال 24/7 على الـ VPS، حتى لو قفلت جهازك. كل ما حوت يشتري meme coin، هتوصلك إشعار على تيليجرام.

🔟 التحكم في البوت (أوامر)

كل ما ترجع للـ VPS، تقدر تتحكم في البوت بالأوامر دي:

# الدخول للفولدر
cd ~/whale-bot

# استخدام سكريبت deploy.sh (الأسهل)
./deploy.sh status        # حالة البوت
./deploy.sh logs          # شوف الـ logs live
./deploy.sh restart       # ريستارت
./deploy.sh stop          # وقف البوت
./deploy.sh start         # تشغيل تاني
./deploy.sh update        # تحديث من GitHub
./deploy.sh uninstall     # حذف كامل

# أو استخدام docker compose مباشرة
docker compose ps          # حالة
docker compose logs -f     # logs
docker compose restart     # ريستارت
docker compose down        # وقف
docker compose up -d       # تشغيل

🔁 تحديث الكود بعد ما ترفع تعديلات على GitHub

# على الـ VPS
cd ~/whale-bot
./deploy.sh update

# أو يدوياً
git pull
docker compose up -d --build

1️⃣1️⃣ مشاكل شائعة وحلولها

المشكلة 1: Authentication failed عند الـ git clone

السبب: الـ Token غلط أو منتهي.

الحل:

  • تأكد إن الـ Token صحيح ومنسوخ بالكامل
  • لو منتهي، روح على GitHub → Settings → Developer settings → Personal access tokens → جدد الـ token
  • تأكد إن الـ token فيه scope repo

المشكلة 2: docker compose مش موجود

الحل:

# تثبيت docker compose plugin
sudo apt install -y docker-compose-plugin

# أو لو مش شغال
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

المشكلة 3: permission denied على deploy.sh

الحل:

chmod +x deploy.sh
./deploy.sh

المشكلة 4: البوت شغال بس مفيش إشعارات

الحل:

# شوف الـ logs
docker compose logs -f whale-bot-solana

# لو فيه أخطاء في Telegram
# - تأكد إن TELEGRAM_CHAT_ID صحيح
# - تأكد إن TELEGRAM_BOT_TOKEN صحيح
# - ابعت /start للبوت الأول على تيليجرام

# لو فيه أخطاء في RPC
# - البوت هيستخدم public RPC تلقائياً

المشكلة 5: Docker requires sudo

الحل:

sudo usermod -aG docker $USER
newgrp docker
# أو اعمل logout و login تاني

المشكلة 6: البوت وقع بعد الإعادة

الحل: استخدم systemd (تشغيل تلقائي بعد الإعادة):

sudo nano /etc/systemd/system/whale-bot.service
[Unit]
Description=Whale Bot
After=docker.service
Requires=docker.service

[Service]
Type=oneshot
RemainAfterExit=yes
WorkingDirectory=/home/ubuntu/whale-bot
ExecStart=/usr/bin/docker compose up -d
ExecStop=/usr/bin/docker compose down
TimeoutStartSec=0

[Install]
WantedBy=multi-user.target
sudo systemctl daemon-reload
sudo systemctl enable whale-bot
sudo systemctl start whale-bot
